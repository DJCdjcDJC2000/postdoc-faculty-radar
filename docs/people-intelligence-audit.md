## 已识别风险

1. 旧构建把人物和课题组原始 JSON 直接透传，没有标准化、质量门槛或字段级证据。
2. 旧公开输出包含个人化匹配、可学习点、风险和产业可行性分数。已增加实体级公开清洗，最终 canonical public DTO 仍要采用字段白名单。
3. 旧人物详情是抽屉，没有可刷新、可深链、可导出的独立档案路由。
4. 旧首屏加载完整 `site.json` 并一次渲染全部页面，无法承载 180 个深档案及论文数据。
5. 产业人物虽然数量达到 51，但证据高度复用，约 12 位直接相关性偏弱；最终仍需替换和补充。
6. 原公开 GitHub Actions 构建 private fixture。已从公开 CI 中移除，私有数据改由独立私有仓库和鉴权服务管理。

## 已建立的门槛

- `config/people-intelligence.json`：数量、质量、隐私、更新周期和招聘信号契约。
- `config/academic-identities.json`：人工核验身份映射。
- `config/venue-taxonomy.json`：5 个赛道、3 个层级、30 个官方来源 venue。
- `scripts/lib/academic-intelligence.mjs`：统一档案、质量评估、招聘信号和总览。
- `scripts/lib/venue-classification.mjs`：venue 分类和 work family 去重。
- 独立私有仓库：个人画像、私有状态 schema 和安全模型。

## 最终完成证据

目标只有在以下证据全部存在时才能标记完成：

- 公开数据实际达到 50 个课题组、80 位青年学者和至少 50 位产业人物，不只是在配置中声明目标。
- 每个公开档案通过官方身份、书目来源、研究变化、论文指标、代表作、证据和更新时间硬门槛。
- 所有独立档案路由、筛选、2-5 人对比、网络、导出和移动端行为经过真实浏览器验证。
- 私有站通过飞书单用户登录，未登录无法读取任何私有 API 或静态数据。
- 公共仓库、Actions 日志、GitHub Pages、Vercel 响应和构建产物通过私有数据 canary 扫描。
- 周/月/季/年更新、DeepSeek 增量分析、审核队列和飞书提醒经过至少一轮真实运行。
- GitHub Pages、大陆优化入口、Vercel 公开主站和私有站均成功部署并从外网验证。
